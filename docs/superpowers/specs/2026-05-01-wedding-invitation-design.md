# Свадебное приглашение Екатерины и Никиты — Spec

**Статус:** утверждено пользователем (брейншторм пройден).
**Дата:** 2026-05-01.
**Цель:** одностраничный сайт-приглашение для свадьбы 04.07.2026 в Волгограде. Аналог по структуре — `https://06-06-26.xn--p1ai`. Стилистика — взятая с бумажного приглашения (светлая, ботаническая, пастельная палитра).

## Технологический стек

- **Astro** (output: static) — компонентный фреймворк для статичных сайтов.
- **TypeScript** — для скриптов и контент-модели.
- **Vanilla CSS** с CSS-переменными — без Tailwind, без CSS-in-JS.
- **Google Fonts** — Pinyon Script (курсив), Cormorant Garamond (заголовки), Inter (текст).
- **Хостинг:** GitHub Pages с автодеплоем через GitHub Actions.

### RSVP (Google Forms)

Поля формы:

## Поведение и логика

### Таймер обратного отсчёта

```ts
const target = new Date(wedding.date.iso).getTime()
setInterval(() => {
  const diff = target - Date.now()
  if (diff <= 0) showCelebration()
  else updateDigits(...)
}, 1000)
```

Без сторонних библиотек, только нативный `Date`.

### Музыка

- `<audio>` скрытый, `loop`, `volume = 0.4`.
- Через 4 секунды после загрузки — попап «Включить?» с кнопками Да / Не сейчас.
- При согласии — `audio.play()` (после клика, чтобы не блокировал autoplay-policy браузеров).
- `localStorage`: `music_pref = 'on' | 'off' | 'asked-no'`. При повторных визитах попап не показывается, если уже отвечал.
- Floating-кнопка ♪ в правом нижнем углу — toggle play/pause всегда.

### RSVP-форма

#### Состояние кнопки «Отправить»

- По умолчанию **disabled**.
- Включается только когда:
  - имя не пустое (после `trim`)
  - выбран один из radio-вариантов

```ts
const validate = () => {
  button.disabled = !(nameInput.value.trim() && [...radios].some(r => r.checked))
}
nameInput.addEventListener('input', validate)
radios.forEach(r => r.addEventListener('change', validate))
```

#### Антиспам

- **Honeypot**: скрытое поле `<input name="phone_alt" tabindex="-1" autocomplete="off">`. Если заполнено — сабмит «успешен» визуально, но в Google ничего не идёт.
- **Капча**: математическое выражение («Сколько будет 3 + 7?») со случайными числами, проверяется при сабмите.

#### Отправка

```ts
async function onSubmit(e) {
  e.preventDefault()
  if (!isHoneypotEmpty() || !isCaptchaCorrect()) return

  button.disabled = true
  await fetch(rsvp.formActionUrl, {
    method: 'POST',
    mode: 'no-cors',
    body: new URLSearchParams({
      [rsvp.fields.name]: nameInput.value.trim(),
      [rsvp.fields.attendance]: getAttendanceValue(),
    }),
  })
  showBanner(getMessageFor(getAttendanceValue()))
  form.reset()
  regenerateCaptcha()
  validate()
  setTimeout(hideBanner, 5000)
}
```

#### Поведение после отправки

- Над формой появляется баннер с сообщением (success или decline).
- Форма **сбрасывается** (имя, radio, капча).
- Кнопка возвращается в disabled.
- Через 5 секунд баннер плавно исчезает.
- **Никакой блокировки повторных отправок** — гость может подтвердить нескольких людей последовательно.

#### Edge cases

- Двойной клик: кнопка дисейблится сразу после первого клика.
- Сетевая ошибка: повтор fetch один раз через 2 секунды.
- При совсем неудаче — короткое сообщение «Что-то пошло не так. Попробуйте ещё раз или обновите страницу.»

### Меню

```js

export default defineConfig({
  site: 'https://<user>.github.io',
```

- Триггер: push в `main` или ручной запуск.
- Шаги: checkout → setup-node 20 → `npm ci` → `npm run build` → upload artifact → deploy-pages.
- Permissions: `pages: write`, `id-token: write`.

### Кастомный домен (когда будет куплен)

1. DNS: A-записи на 4 IP GitHub Pages (apex) или CNAME на `<user>.github.io` (поддомен).
2. Файл `public/CNAME` с доменом.
3. Settings → Pages → Custom domain + Enforce HTTPS.

### Внешние секреты

- **Никаких секретов в репо или бандле.** RSVP идёт на публичный URL Google Forms.
- Если в будущем добавим Telegram-бот — токен через GitHub Secrets + build-time env, или через Cloudflare Worker.

## Заглушки на старте

Решение зафиксировано: на момент первого деплоя реальные фото и аудио ещё не готовы. Используем заглушки:

- **Hero-фото:** SVG-композиция с ботаническими элементами (эвкалипт, пампасная трава, мелкие розы) в палитре дресс-кода — стилистически соответствует бумажному приглашению. Размер 1:1, 480×480.
- **Map-секция:** сразу используется встроенная Яндекс.Карта (iframe) с маркером ресторана «Фьюжн» — фото не требуется и заглушка не нужна.
- **Фото для пунктов программы:** маленькие круглые SVG-иконки (свеча, кольца, бокалы, торт) или просто кружки с цветом палитры — без фото.
- **Фоновая музыка:** аудио-элемент создаётся в DOM, но `src` — пустая строка. Кнопка ♪ и попап функционируют, но при клике «Да» показывается toast «Музыка появится позже» вместо реального воспроизведения. После замены mp3 в `public/audio/background.mp3` всё включается без правок кода.

Замена заглушек на реальные ассеты — без изменения структуры или кода. Только файлы в `public/`.
